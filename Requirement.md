# Requirements.md  
## 物品管理アプリ（macOS）

### 0.　コード書き方
Swift のコード規約は、Apple の公式ガイドライン + Swift コミュニティ標準の二つに準拠する

### 1. 概要  
本アプリは、ユーザーが所有する **私物全般を管理するための macOS アプリ** である。  
SwiftUI + SwiftData を用いてローカルデータベースとして動作し、写真機能や外部同期を伴わない軽量な管理ツールとする。

---

## 2. プラットフォーム  
- macOS アプリ（Swift / SwiftUI / SwiftData）
- ローカル DB（iCloud 同期なし）

---

## 3. 管理対象  
ユーザーが所有する **すべての私物（書籍、衣類、ガジェット、家具、日用品など）** を統一的に管理する。

---

## 4. UI 仕様  
- **2 ペイン構造（NavigationSplitView）**
  - 左：一覧（検索、フィルタ、ソート）
  - 右：詳細（閲覧・編集）
- macOS アプリとして自然な UI 操作性を重視する

---

## 5. データモデル（項目）  

### ■ Item（物品）
**基本情報（必須）**  
- 名前（必須）  
- カテゴリ（Category 参照）  
- 所有場所（Location 参照）  
- メモ  
- タグ（Tag 多対多）

**詳細情報**  
- ブランド / メーカー  
- 型番  
- シリアル番号  
- 購入日  
- 購入店  
- 購入価格  
- 保証期限  
- サイズ・寸法  
- 重量  
- カラー  
- 状態（Condition：固定リストから選択）  
- 付属品  
- 消耗品の交換時期  
- リンク（URL）

**運用情報（自動管理）**  
- 登録日（自動付与）  
- 最終更新日（自動更新）  
- 廃棄／売却フラグ  
- 廃棄／売却日

---

## 6. マスタ管理

### ■ Category（カテゴリ）
- 階層なし（フラット）
- CRUD 可能（作成／編集／削除）

### ■ Location（所有場所）
- 階層なし  
- CRUD 可能  
- 例：自宅／リビング／クローゼット

### ■ Tag（タグ）
- 階層なし  
- 多対多  
- CRUD 可能

### ■ Condition（状態）
- 固定リスト（編集不可）
  - 新品  
  - 良好  
  - 普通  
  - 劣化  
  - 壊れている  

---

## 7. 非機能要件
- ローカルのみで動作（ネットワーク不要）
- SwiftData による軽量・高速なデータアクセス
- 写真データは扱わないため DB が肥大化しない
- シンプルで操作しやすい UI
